明经通道 Visual LISP 函数  
vlisp-compile
 

将 AutoLISP 源代码生成为 FAS 文件

(vlisp-compile 'mode filename [out-filename])

注意 必须打开 Visual LISP IDE 以保证 vlisp-compile 能正常运行。

参数

mode

编译器模式，可为下列符号之一：

st 标准编译模式

lsm 优化但不直接链接

lsa 优化且直接链接

filename

字符串，表示 AutoLISP 源文件。如果源文件位于 AutoCAD 支持文件搜索路径中，则在指定文件名时可以省略路径。如果省略扩展名，则假定其为 .lsp。

out-filename

字符串，表示生成后的输出文件。如果没有指定输出文件，vlisp-compile 设定输出文件与原文将同名，但将扩展名替换为 .fas。

请注意如果指定了输出文件名，但没有指定I/O文件的路径，则 vlisp-compile 将输出文件置于 AutoCAD 安装目录下。

返回值

如果编译成功则返回 T，否则返回 nil。

示例

假定 yinyang.lsp 位于 AutoCAD 支持文件搜索路径中，则下面的命令编译该程序：

_$ (vlisp-compile
'st "yinyang.lsp")

T

输出文件名为 yinyang.fas，与源文件位于同一目录。

下面的命令生成 yinyang.lsp 并将输出文件命名为 GoodKarma.fas：

(vlisp-compile 'st "yinyang.lsp" "GoodKarma.fas")

请注意上一命令中的输出文件位于 AutoCAD 安装目录中，而不是 yinyang.lsp 所在的目录。下面的命令生成 yinyang.lsp 并将输出文件置于 c:\my documents 目录：

(vlisp-compile
'st "yinyang.lsp" "c:/my documents/GoodKarma")

最后一个样例中指定了源文件的完整路径：

(vlisp-compile
‘st "c:/program files/<AutoCAD installation directory>/Sample/yinyang.lsp")

输出文件名为 yinyang.fas，与输入文件位于同一目录。
